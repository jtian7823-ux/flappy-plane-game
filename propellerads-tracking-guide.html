🌐 落地頁網址

https://jtian7823-ux.github.io/flappy-plane-game/

🔗 投放連結格式（必帶參數）

https://jtian7823-ux.github.io/flappy-plane-game/?utm_prid={SUBID}

參數說明

參數 必填 說明
utm_prid ✅ 必填 PropellerAds 的 Subid macro，系統投放時自動替換為真實訪客 ID

PropellerAds 後台填法

在廣告設定的 Landing Page URL 欄位填入：

https://jtian7823-ux.github.io/flappy-plane-game/?utm_prid={SUBID}

{SUBID} 是動態 macro，不需要手動替換，PropellerAds 系統投放時自動填入。

⚡ 追蹤流程

1
用戶點擊廣告，進入落地頁（URL 帶 utm_prid=xxxxx）
2
GTM 自動讀取 utm_prid，存入 Cookie（prop_visitor_id）
3
用戶完成註冊 → GTM 觸發 Registration tag → 回傳轉換給 PropellerAds
4
用戶完成首存 → GTM 觸發 Deposit tag（goal=2）→ 回傳轉換給 PropellerAds

🧪 測試方法

用測試 subid 開啟落地頁：

https://jtian7823-ux.github.io/flappy-plane-game/?utm_prid=test123

驗證步驟

  1. 打開瀏覽器 DevTools（F12）
  2. 前往 Application → Cookies → 找到落地頁網域
  3. 確認 prop_visitor_id=test123 存在 ✅
  4. 觸發註冊事件 → Network tab 搜尋 conversion.php → 確認 GET request 含 visitor_id ✅
  5. 觸發首存事件 → 確認 URL 含 goal=2 ✅

⚠️ 注意事項

  • utm_prid 不能省略，沒有這個參數就無法追蹤轉換
  • Cookie 有效期 30 天，用戶在 30 天內完成轉換都會被記錄
  • 同一個 subid 重複進入頁面不影響，Cookie 會更新為最新值
